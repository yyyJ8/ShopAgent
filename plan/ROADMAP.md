# OZON 数据分析 Agent — 关键节奏

> 从零搭建，基于真实 OZON 数据（2 店铺 / ~100 商品）。
> 不做 Text-to-SQL，做**语义化数据工具 + LangGraph 智能分析 Agent**。

---

## 里程碑

| 阶段 | 时间 | 目标 | 产出 |
|---|---|---|---|
| **MCP Server** | 第 1–2 周 | 语义化数据工具层 | 4 个 MCP Tool（sales / ad / finance / inventory），内部固定 SQL |
| **LangGraph Agent** | 第 1–2 周 | 智能分析编排 | 理解问题 → 选工具 → 传参 → 拿数据 → 交叉分析 → 异常检测 → 运营建议 |
| **评测闭环** | 第 3 周 | 可量化质量 | 20 道测试题 + 标准答案 + 准确率指标 |
| **配置 & 完善** | 第 4 周 | 业务口径 + 可观测 + 交付 | 配置文件注入、Streamlit 图表、多轮对话、README |
| **面试模式** | 第 5 周起 | 能讲透 | 项目复盘、博客/笔记、模拟面试 |

## 技术架构（一张图）

```
用户自然语言
    ↓
LangGraph Agent（你写）
  ├── 理解问题
  ├── 选择工具 + 传参
  ├── 拿数据 → 交叉分析
  ├── 异常检测（规则打标记 + LLM 归因）
  └── 运营建议
    ↓
MCP Server（你写）
  ├── get_sales_data(date_range, sku_id?)
  ├── get_ad_data(date_range, sku_id?)
  ├── get_finance_data(date_range)
  └── get_inventory_data()
    ↓
PostgreSQL（固定 SQL，与 BI 看板同款）
```

## 三层递进

| 层级 | 能力 | 示例问题 |
|---|---|---|
| **查数层** | 自然语言 → 工具调用 → 数据解读 | "最近 7 天销量 Top 10 是哪些？" |
| **异常检测层** | 自动发现销量暴跌 / ROI 异常 / 库存预警 | "有没有哪个 SKU 销量突然跌了 30% 以上？" |
| **运营建议层** | 异常归因 + 可执行建议 | "为什么这个品跌了？建议怎么处理？" |

---

## Harness 三层映射

| Harness 层 | 实现 | 面试一句话 |
|---|---|---|
| **控制层** | MCP 工具边界 + 固定 SQL + 参数校验 | Agent 碰不到 SQL，只能调预设工具——防错而非试错 |
| **记忆层** | 配置文件注入业务口径（动销率/转化率/异常阈值等） | 业务知识沉淀在配置里，不在 prompt 里硬编码 |
| **编排层** | MCP 语义化工具 + LangGraph 状态机 | 异构系统封装成统一工具，Agent 只做路由和推理 |

## 评测闭环

```
20 道测试题 + 标准答案
        ↓
   跑 Agent
        ↓
  对比答案 → 准确率 X%
        ↓
  错的记下来 → 改 prompt / 改工具 / 改配置
        ↓
   再跑一轮 → 准确率涨了？
```

简历一句话：「分析准确率 X%，有完整评测闭环。」

## 异常检测策略（规则 + LLM 混合）

| 环节 | 谁做 | 为什么 |
|---|---|---|
| **检测**（是否异常） | 规则（阈值 / 同比 / 环比） | 确定性、可复现、零幻觉 |
| **归因**（为什么异常） | LLM 推理 | 发挥模型推理能力，多维度交叉分析 |
| **建议**（怎么办） | LLM 生成 | 需要业务知识 + 自然语言表达 |

---

## 关键决策记录

| 决策 | 选什么 | 为什么不选另一个 |
|---|---|---|
| **不做 Text-to-SQL** | 语义化工具 + 固定 SQL | BI 已用固定 SQL 覆盖高频需求；Text-to-SQL 工业界准确率低、安全风险大 |
| **不用 RAG** | 配置文件注入业务口径 | 仅二十来条口径，RAG 过度工程；RAG 能力由 RAGWISE 背书 |
| **Agent 不碰 SQL** | MCP 工具封装 | 安全边界清晰，Agent 只管"调哪个工具+传什么参" |
| **规则检测 + LLM 归因** | 混合策略 | 检测要可靠（规则），归因要智能（LLM），各取所长 |
